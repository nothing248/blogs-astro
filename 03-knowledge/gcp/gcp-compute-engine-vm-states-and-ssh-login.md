## 1. 核心定位

**Google Compute Engine (GCE)** 是 GCP 平台提供的传统 IaaS 服务，即按需提供可高度定制的虚拟服务器（VM）。

---

## 2. 虚机生命周期与状态计费

理解虚机不同操作状态对数据持久化与计费的影响至关重要：

| 操作状态 | 系统影响 | 数据保留情况 | 计费状态 |
| :--- | :--- | :--- | :--- |
| **停止 (Stop)** | 彻底关机。 | **保留**：永久磁盘与元数据。<br> **丢失**：内存数据与临时分配的外部 IP。 | **免除** vCPU 与内存费用（仅对挂载的磁盘持续计费）。 |
| **暂停 (Suspend)** | 系统挂起（类似电脑休眠）。 | **保留**：内存状态、永久磁盘。<br> **丢失**：临时外部 IP。 | 对保存内存状态所需的存储空间 **收取留存费用**。 |
| **重置 (Reset)** | 强行冷重启（类似物理机按 Reset 键）。 | **保留**：永久磁盘。<br> **丢失**：内存数据被擦除。 | 虚机仍在运行生命周期内，**正常产生计算与存储全额计费**。 |

---

## 3. 远程连接 (SSH) 排障与避坑

### 3.1. OS Login 冲突限制

GCP 推荐使用 OS Login 统一管理团队虚机权限，但这会与传统的 SSH Key 产生互斥机制。

- 若实例或项目的元数据中开启了 `enable-oslogin = TRUE`，则系统将 **拒绝识别** 通过元数据手工配置的传统 SSH 密钥。
- **排障**：如必须使用独立私钥直连服务器，请确保该标识被设为 `FALSE`。

### 3.2. 家目录权限错误

- 若连接被服务器秒拒且无明显网络问题，应排查 `/home/[user]` 目录的写权限。若当前登录用户的家目录对其自身或其他组缺少正确的写权限，GCP 底层的认证脚本将导致 SSH 进程握手失败。
