## 1. 丢失修改

在 Git 中，“删除工作区但未暂存的文件”需要根据文件的状态分为两种情况处理：**已追踪的文件（修改了内容）和未追踪的新文件（刚创建的文件）**。

---

### 1.1. 情况一：丢弃“已追踪”文件的修改

如果你修改了已经存在的文件，想把它们恢复到和上次提交一模一样的状态（即丢弃当前改动）：

#### 1.1.1. 恢复单个或多个文件 (推荐方式)

使用 `git restore`（Git 2.23+ 引入，语义更明确）：

Bash

```
# 恢复特定文件
git restore <文件名>

# 恢复当前目录下所有修改
git restore .
```

#### 1.1.2. 使用旧版命令

如果你习惯经典命令或 Git 版本较低：

Bash

```
# 恢复特定文件
git checkout -- <文件名>

# 恢复所有文件
git checkout .
```

---

### 1.2. 情况二：删除“未追踪”的新文件 (Untracked Files)

如果你新创建了一些文件，但还没运行过 `git add`，这些文件不属于 Git 的版本管理。`checkout` 或 `restore` 对它们无效，必须使用 `git clean`。

#### 1.2.1. 先预览将要删除的文件 (安全第一)

为了防止误删，先看看哪些文件会被清理：

Bash

```
git clean -n
```

#### 1.2.2. 确认删除

Bash

```
# 删除当前目录下所有未追踪的文件
git clean -f

# 删除未追踪的文件以及空的文件夹
git clean -fd

# 删除所有未追踪的文件，包括被 .gitignore 忽略的文件（慎用）
git clean -fx
```

---

### 1.3. 情况三：一次性清空所有修改（最彻底）

如果你想让整个工作区瞬间回到上次提交的状态，**包括** 丢弃已追踪文件的修改、**并且** 清理掉暂存区里的东西（但不会删除未追踪的新文件）：

Bash

```
git reset --hard HEAD
```

_注意：如果你还想顺便把未追踪的新文件也删了，需要组合使用：_

Bash

```
git reset --hard HEAD && git clean -fd
```

---

### 1.4. 总结对照表

|**目标**|**命令**|**备注**|
|---|---|---|
|**丢弃已追踪文件的改动**|`git restore <file>`|恢复到上次 commit 的内容|
|**删除新创建的杂乱文件**|`git clean -f`|物理删除未 add 的新文件|
|**删除新创建的文件夹**|`git clean -fd`|连同新目录一起删除|
|**全部推倒重来**|`git reset --hard`|慎用！会丢失所有未提交的改动|

**💡 安全建议**：如果你不确定这些修改以后是否还会用到，最稳妥的方法不是删除，而是使用 `git stash` 将它们暂时存入堆栈，等确定不要了再清空。
