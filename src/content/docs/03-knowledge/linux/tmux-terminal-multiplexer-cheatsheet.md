## 1. 工具定位

**Tmux** 是一个强大的终端复用器。它的核心价值在于：允许用户在单个终端窗口内创建多个互相独立的会话（Session）与窗格（Pane）；且当网络断开或人为关闭 SSH 连接时，**Tmux 内运行的进程（如模型训练、长耗时下载）仍能在后台持续运行**。

*安装（CentOS）：`yum install tmux` / (Ubuntu)：`apt install tmux`*

---

## 2. 核心操作与快捷键字典

> [!note] 快捷键前缀
> Tmux 的所有内部操作指令都必须先按下默认前缀键组合 **`Ctrl + b`**，松开后，再按下对应的字母键。

### 2.1. 会话级管理 (Session)

会话是最大的隔离层级，常用于按项目分类。

- **终端外命令**：
  - `tmux new -s [name]`：新建指定名称的会话。
  - `tmux ls`：列出当前所有后台会话。
  - `tmux attach -t [name/id]`：重新接入之前挂起的会话。
  - `tmux kill-session -t [name/id]`：强行杀死会话及其中进程。
- **终端内快捷键 (需先按 `Ctrl+b`)**：
  - `d`：**Detach (极其常用)**，暂时分离当前会话，将其退至后台运行。
  - `s`：以列表模式查看并切换会话。
  - `$`：重命名当前会话。

### 2.2. 窗口级管理 (Window)

相当于浏览器的标签页，在同一个会话中开启多个工作区。

- `c`：(Create) 创建一个新窗口。
- `[数字 0-9]`：直接切换到对应的窗口编号。
- `w`：(Windows) 列出所有窗口并进行选择切换。
- `,`：重命名当前窗口，方便记忆。

### 2.3. 窗格级管理 (Pane)

将当前窗口进一步切分成多个子画面（分屏）。

- `%`：**垂直（左右）** 划分当前窗格。
- `"`：**水平（上下）** 划分当前窗格。
- `[方向键]`：光标在不同的窗格间移动。
- `x`：关闭并销毁当前所在的光标窗格。
- `!`：将当前窗格“弹出”，成为一个独立的全屏窗口。

## 3. 高频配置：开启鼠标滚屏支持

默认情况下，在 Tmux 中直接使用鼠标滚轮会翻阅系统终端的历史记录，而不是 Tmux 内程序的历史输出。必须开启鼠标模式。

**持久化生效 (推荐)**：
执行以下命令将配置写入用户根目录并重启服务：

```bash
echo "set -g mouse on" >> ~/.tmux.conf
tmux kill-server && tmux
```

**临时生效机制**：

1. 进入 Tmux 会话内部。
2. 按下前缀键 `Ctrl + b`，松开。
3. 输入冒号 `:` 进入底部的命令行模式。
4. 输入 `set -g mouse on` 并回车确认。
