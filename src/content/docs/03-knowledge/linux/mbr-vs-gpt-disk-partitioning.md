在磁盘分区领域，**MBR (Master Boot Record)** 和 **GPT (GUID Partition Table)** 是决定操作系统如何组织和访问硬盘数据的两种不同标准。

简单来说，MBR 是“老前辈”，兼容性强但局限多；GPT 是“现代派”，是大容量硬盘和新系统环境下的标配。

---

## 1. MBR (主引导记录)

MBR 诞生于 1983 年，由于其悠久的历史，几乎支持所有操作系统。

- **工作原理**：分区信息存储在磁盘的第一个扇区（512 字节）。
- **核心限制**：
  - **容量限制**：最大仅支持 **2 TB** 的磁盘。如果硬盘是 4 TB，使用 MBR 只能识别前 2 TB。
  - **分区数量**：最多支持 **4 个主分区**。如果需要更多分区，必须将其中一个设为“扩展分区”，再在里面创建“逻辑分区”。
  - **脆弱性**：分区表只存储在一处。如果该扇区损坏，整个磁盘的数据将难以读取。
- **启动模式**：必须配合 **Legacy BIOS** 模式引导启动。

---

## 2. GPT (GUID 分区表)

GPT 是伴随 UEFI 出现的新一代分区标准，解决了 MBR 的局限性痛点。

- **工作原理**：使用全局唯一标识符 (GUID) 来标识每个分区。
- **核心优势**：
  - **超大容量**：理论支持最大 **18 EB** ($1 \text{ EB} = 1024 \text{ PB} = 1,048,576 \text{ TB}$) 的磁盘。
  - **分区数量**：Windows 环境下默认支持多达 **128 个主分区**，不再需要额外的扩展/逻辑分区。
  - **高可靠性**：GPT 在磁盘末尾备份了分区表。如果开头的分区表损坏，系统可以自动提取备份修复。同时它还包含循环冗余校验 (CRC) 以检测数据完整性损坏。
  - **兼容防御**：GPT 保留了一个“保护性 MBR” (Protective MBR)，防止只支持 MBR 的旧工具误认为磁盘未分区而将其初始化清空。
- **启动模式**：必须配合 **UEFI** 模式引导启动。

---

## 3. 核心对比表

| 特性 | MBR (旧标准) | GPT (新标准) |
| :--- | :--- | :--- |
| **最大磁盘容量** | 2 TB | 18 EB (几乎无限) |
| **分区限制** | 4 个主分区 | 128 个主分区 (Windows) |
| **启动架构** | Legacy BIOS | UEFI |
| **操作系统支持** | 几乎所有 (包括 WinXP/Win7 32 位) | Win8/10/11, macOS, 近年 Linux |
| **安全性** | 分区表易损坏且无备份 | 具有冗余备份和 CRC 校验 |

## 4. 分区转化

### 4.1. 查看分区类型

输入以下命令查看磁盘分区信息：

```bash
sudo parted -l
```

在输出结果中寻找 **Partition Table** 一栏：

- 如果显示 `msdos`，说明是 MBR 格式。
- 如果显示 **`gpt`**，说明是 GPT 格式。

### 4.2. 方案一：新硬盘初始化（最简单）

如果准备给挂载的全新数据盘（例如树莓派外接盘）或重装系统时初始化 GPT 格式：

#### 4.2.1. 使用命令行工具 `fdisk` (交互式)

假设磁盘设备名是 `/dev/sda`（请务必通过 `lsblk` 确认实际名称）：

1. 执行 `sudo fdisk /dev/sda`。
2. 输入 `g`：这会将磁盘转换为 **GPT** 格式。
3. 输入 `n`：新建分区。
4. 输入 `w`：保存分区并退出。

---

### 4.3. 方案二：无损转换（将现有的 MBR 系统盘转为 GPT）

如果已在 USB 盘或系统盘中安装好了系统（当前为 MBR 分区），但想在不重装、不丢失数据的情况下转换为 GPT（例如需要支持 2TB 以上容量或启用安全的 UEFI 引导）：

在 Linux 环境下，可以使用 `gdisk` 工具。

1. **安装工具**：

```bash
sudo apt update
sudo apt install gdisk
```

1. **开始转换**：

```bash
sudo gdisk /dev/sda  # 请确保 sda 是你的目标盘符
```

1. **转换指令操作**：
   - `gdisk` 会自动检测到 MBR 分区，并询问是否转换。
   - 输入 `w` (写入命令)。
   - 输入 `Y` 确认执行操作。
   - 此时分区表已变为 GPT 格式。
