## 1. 分布式系统核心治理维度

构建分布式系统需在 CAP 定理（一致性、可用性、分区容错性）之间寻找平衡点。

### 1.1. 数据一致性与事务

- **最终一致性**：通过异步补偿、消息队列保证数据在“最终”达到一致。
- **分布式事务**：涉及 2PC（二阶段提交）、TCC（Try-Confirm-Cancel）及基于消息表的事实事务。

### 1.2. 服务发现与稳定性

- **服务注册与发现**：动态管理服务实例 IP，集成 Consul/Nacos。
- **负载均衡**：四层（L4）与七层（L5）流量分发策略。
- **熔断与降级**：通过断路器隔离故障节点，提供有损服务保证核心链路可用。

### 1.3. 高并发治理工具

- **分布式锁**：基于 Redis (Redlock) 或 Zookeeper 解决资源竞争。
- **幂等性设计**：通过去重表、SourceID 保证请求重复执行的结果唯一。

### 1.4. 观测与分析

- **分布式链路追踪**：分析跨服务的请求延迟。
- **日志聚合**：解决由于节点分散导致的日志查询困难问题（ELK 架构）。

> [!warning] 核心陷阱
> 分布式系统最昂贵的成本往往不在于开发，而在于网络分区（Network Partition）带来的各种极端异常处理逻辑。
