## 2. Rime 生态系统的基本结构

首先，理解 Rime（中 ime/librime）是一个 **输入法框架或核心程序库**，它定义了输入法的规则、算法和配置文件格式。它本身不直接提供给用户使用，需要客户端（Frontend）来实现用户界面。

---

### 2.1. Weasel (小狼毫)

**角色：操作系统客户端 / 载体**

|特点|描述|
|---|---|
|**定义**|**Rime 在 Windows 操作系统上的官方客户端程序**。|
|**功能**|负责将 Rime 核心（librime）的输入逻辑以图形界面的方式呈现给用户，包括输入框、选词窗、系统集成等。|
|**关系**|如果您在 Windows 上使用 Rime，那么您安装的就是 **Weasel**。它是您与 Rime 引擎交互的“界面”。|

---

### 2.2. Plum (东风破)

**角色：配置管理工具 / 方案仓库**

| 特点 | 描述 |
| ------ | ----------------------------------------------------------------------------------------------- |
| **定义** | Rime 官方提供的 **配置方案管理工具和配置仓库**（GitHub 上的 `rime/plum`）。 |
| **功能** | 它的主要作用是帮助用户 **安装、更新或管理** 各种 Rime 输入方案（如五笔、双拼、颜文字等）和用户词库。 |
| **关系** | Plum 本身 **不是输入法**，而是一个 **配置安装器**。它能简化用户下载和部署复杂的 Rime 配置文件的过程。用户通过 Plum 可以一键获取并应用 Rime 社区的各种定制化方案。 |

---

### 2.3. Rime-ice (Rime 冰)

**角色：第三方输入方案 / 配置集**

|特点|描述|
|---|---|
|**定义**|这是一个 **高度定制化、优化和美化** 的 **Rime 输入法配置方案集合**。它是一个“配置包”。|
|**功能**|Rime-ice 通常会整合多种流行的输入法配置（如特定的拼音方案、词库、快捷键设置、外观主题等），并进行优化，让用户能快速获得一个 **开箱即用、体验优秀** 的 Rime 环境。|
|**关系**|Rime-ice 必须运行在 Rime 客户端（如 **Weasel**、Squirrel 等）之上。您可以利用 **Plum** 工具，将 **Rime-ice** 这一套配置方案 **安装和部署** 到您的 **Weasel** 客户端中。|

---

### 2.4. 总结和关系图

可以把它们的关系比喻为：

1. **Rime（中州韵）：** **发动机**（核心技术和原理）。
2. **Weasel（小狼毫）：** **汽车外壳**（Windows 上的用户界面和客户端）。
3. **Plum（东风破）：** **工具箱**（用于安装和升级配置文件的工具）。
4. **Rime-ice（Rime 冰）：** **豪华内饰**（一套优化的、现成的配置文件，决定了您的输入体验）。

**流程：**

用户首先在 Windows 上安装 **Weasel** → 然后，使用 **Plum** 工具 → 将 **Rime-ice** 这套优秀的 **配置方案** → 部署到 **Weasel** 客户端中 → 最终获得一个 **高度优化和美化** 的 Rime 输入体验。

## 3. 配置信息

### 3.1. 核心层：通用设置

这部分配置影响所有或大部分输入方案，定义了输入法的底层行为、通用快捷键和模式切换。

|文件名|作用|对应配置项|
|---|---|---|
|**`default.yaml`**|**通用配置**（所有方案的默认设置）|基础快捷键（如中英切换）、模式切换定义、共享词典引用、配置结构版本等。|
|**`default.custom.yaml`**|**全局补丁**|**用户自定义** 的全局设置，用于修改或覆盖 `default.yaml` 中的内容。这是最常用的文件之一。|

### 3.2. 前端层：外观和平台特定设置

这部分配置影响输入法的显示外观和平台相关的行为。

|文件名|作用|对应配置项|
|---|---|---|
|**`squirrel.yaml`**|**macOS 前端配置**（鼠须管）|定义输入框、候选词列表的 **外观**（字体、颜色、横/直排）、状态栏显示、特定于 macOS 的行为等。|
|**`weasel.yaml`**|**Windows 前端配置**（小狼毫）|类似 `squirrel.yaml`，定义 Windows 平台上的外观和行为。|

### 3.3. 方案层：特定输入法规则

这部分是定义每一种输入法（如拼音、五笔、双拼）的专属规则。

|文件名|作用|对应配置项|
|---|---|---|
|**`<方案名>.schema.yaml`**|**输入方案主文件**|定义特定输入法的 **核心算法**（`engine`）、编码规则（`speller`）、词典文件引用（`translator`）、方案名称等。例如 `rime_ice.schema.yaml`。|
|**`<方案名>.custom.yaml`**|**方案专属补丁**|针对特定方案进行修改和定制，例如给“雾凇拼音”开启模糊音、调整词频权重等。**这是定制方案（如雾凇拼音）最关键的文件。**|
|**`<词典名>.dict.yaml`**|**词典文件**|存储输入法使用的词汇和词频数据。例如 `rime_ice.dict.yaml`。|

### 3.4. 修改

补丁文件的核心是一个 **`patch:`** 节点，它使用 YAML 路径来定位并修改原始配置文件中的设置。

```
patch:
 <原始配置文件的配置路径>: <新值或操作> 
 <另一个配置路径>: <新值或操作>
```

## 4. 拓展信息

### 4.1. 自定义规则语法

| 操作符 | 作用 | 匹配结果处理 | 转换强度 | 常见用途 |
| ------------ | ------------- | -------------- | ---- | --------------- |
| **`derive`** | **派生** 新的输入码 | **原始码和派生码都有效** | 弱 | 模糊音、多音字容错 |
| **`abbrev`** | **替换** 为完整输入码 | **只使用替换后的码** | 强 | 简拼、双拼、特定输入容错 |
| **`erase`** | **消除** 匹配到的输入码 | 匹配到的部分被 **丢弃** | 最强 | 阻止特定字符输入、特殊键位过滤 |
